Product proposal

Skippy as your second brain, project manager, and agent memory layer

The goal is not to make Skippy another notes app. The goal is to make Skippy the durable memory and coordination system that follows you across Codex, Claude, Hermes, ChatGPT, and any future harness you use.

Second brain Project manager Personal assistant MCP memory layer Source-aware knowledge

The North Star

Skippy should know what matters to you, remember what you are building, capture important context as you work, and make that memory available to any authenticated AI harness through the Skippy MCP.

In plain terms: Skippy should answer "What should Jeff pay attention to?" and also "What does Jeff know, care about, believe, decide, revisit, avoid, and build over time?"

What Skippy Is Today

  • A project and task dashboard.
  • A focus surface for what needs attention now.
  • An MCP-backed system that can ingest from connected sources.
  • A place for source refs, actions, review items, and settings.

What Skippy Should Become

  • A durable personal memory store.
  • A source-aware assistant that can cite why it believes something.
  • A thinking partner that captures decisions, principles, questions, and insights.
  • A shared context layer for many agent harnesses.

The Agent Harness Dream

When you are working in Codex, Claude, Hermes, or another harness, that harness is authenticated to Skippy as you. Skippy watches for durable context, project signals, decisions, preferences, and commitments. Sometimes the harness asks a quick confirmation question, but often it quietly records the right thing with evidence.

1

Work happens in a harness

You discuss code, strategy, relationships, goals, or planning in Codex, Claude, Hermes, or another tool.

2

The harness notices memory signals

It detects a possible project, decision, durable preference, commitment, principle, question, or useful source reference.

3

Skippy decides the right capture path

Clear items can be recorded directly. Ambiguous items can become review candidates. Sensitive or high-impact items can trigger confirmation.

4

The memory becomes reusable

Future harnesses can retrieve it through Skippy MCP, with source refs and a short explanation of why it matters.

What "Second Brain" Means Here

Not enough

  • A pile of notes.
  • A task list with AI summaries.
  • A chat history search box.
  • Uncited answers based on fuzzy memory.

The better target

  • Capture important context automatically and manually.
  • Distill raw input into durable knowledge.
  • Link memory to projects, people, goals, and sources.
  • Resurface relevant context when it can help.

Core Memory Types To Add

Thought

A raw idea, observation, reflection, or captured fragment.

Decision

What was decided, why, when, and what alternatives were rejected.

Principle

A durable rule, preference, taste, or operating belief.

Question

An open inquiry that Skippy should track and revisit.

Insight

A distilled learning that may apply beyond one source item.

Memory

A durable fact about you, your work, relationships, or history.

Skippy-Initiated Interviews

Skippy should be able to run lightweight interviews to fill in the parts of a second brain that automation cannot infer well. These should feel like check-ins, not forms.

  • Project interviews: "What is this project trying to become?"
  • Goal interviews: "What would make this successful by the end of the month?"
  • People interviews: "Who matters here, and what should I remember about them?"
  • Decision interviews: "What did you choose, and why?"
  • Weekly review interviews: "What changed? What should not be forgotten?"

Consent and Autonomy Rules

The agent should be smart enough to capture obvious memory, but the rules need to be explicit so Skippy feels trustworthy.

  • Direct capture is okay for low-risk durable context with clear source evidence.
  • Ask first when creating a major project, changing a priority, or storing sensitive personal context.
  • Send uncertain items to review instead of silently polluting memory.
  • Never store raw private conversations when a processed summary and source ref is enough.
  • Every agent-written memory should explain why it was captured.

Implementation Roadmap

1

Foundation

Add memory tables, source-backed provenance, memory status, confidence, and rubric decision fields.

2

Capture surfaces

Add Memory Inbox, quick capture, direct ingestion rules, and review candidates for uncertain items.

3

Harness integration

Add MCP tools so Codex, Claude, Hermes, and future agents can capture, search, and retrieve context.

4

Interviews and reviews

Add guided interviews, weekly review, project review, and stale-context cleanup.

5

Semantic recall

Add search, context bundles, resurfacing, and source-cited answers across the whole memory graph.

The Short Version

Make Skippy the place where important context becomes durable, source-backed, searchable, and reusable by any harness. Keep projects and tasks, but add memories, decisions, principles, questions, interviews, reviews, and agent-facing MCP tools.

That would turn Skippy from "an app that helps manage work" into "the memory layer that helps every AI you use understand your work."
